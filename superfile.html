1. Document Structure and Head Meta

Every HTML page usually starts with <!doctype html> and then the root <html> element.

Main Parts of an HTML Document

  • <html lang="en"> sets the document language.
  • <head> stores metadata for the browser and search engines.
  • <body> stores the visible content of the page.

Useful Head Elements

  • <meta charset="UTF-8"> supports many characters and symbols.
  • <meta name="viewport"> helps pages behave better on mobile.
  • <title> sets the browser tab title.
  • <meta name="description"> describes the page.
  • <link rel="icon"> sets a favicon.

Example Structure

<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello HTML</h1>
  </body>
</html>

4. Block, Inline, Div, and Span

Block Elements

Block elements usually start on a new line and take up full width.

Examples: div, p, section, article, ul.

Inline Elements

Inline elements usually stay in the same line and only take the space they need. Examples: span, anchor, strong, em.

5. HTML Entities and Symbols

Use entities when you need reserved characters or special symbols.

  • &lt; shows <
  • &gt; shows >
  • &amp; shows &
  • &copy; shows ©
  • &euro; shows €
  • &nbsp; creates a non-breaking space.

7. Semantic HTML

Semantic elements describe the meaning of content instead of using only generic containers.

Common Semantic Tags

  • header for introductory content.
  • nav for navigation links.
  • main for the main content of the page.
  • section for grouped content.
  • article for self-contained content.
  • aside for side content.
  • footer for bottom information.

11. Important Notes and Best Practices

  • Use semantic tags when possible instead of too many generic divs.
  • Always add alt text for meaningful images.
  • Use forms with proper labels for better accessibility.
  • Use tables only for data, not page layout.
  • Use heading order properly, usually one main h1 per page.
  • Use comments only when they add value.
  • Always close elements properly and keep indentation clean.
  • Prefer strong and em when meaning matters.
